Fully Constrained Inline Syntax

Everything happens strictly inside the < > delimiters.

1

The Trailing Flags Approach

Chain components with &&, then append -key:value flags at the end (still inside the tag) to configure the dialog's layout.

<d:msg:Pick player && d:tab:Player -cols:3 -exit:Cancel>
+ Feels exactly like existing CommandPrompter syntax
+ Parser just needs a regex loop to pull -key:value pairs into a Map before splitting by &&
2

The Config Sub-Tag Approach

Dedicate a special cfg sub-tag to hold all the layout settings in its argument brackets.

<d:cfg[cols=3,exit=Cancel] && d:msg:Pick player && d:tab:Player>
+ Keeps arguments neatly contained in standard bracket [] parsing
- Order might be confusing (does cfg have to be first?)
3

The Base Header Approach

Overload the very first tag's key to accept block-level arguments.

<d[cols=3,exit=Cancel]:msg:Pick player && d:tab:Player>
+ Immediately identifies the dialog configuration
- Requires rewriting how the parser identifies the first : and the prompt key
