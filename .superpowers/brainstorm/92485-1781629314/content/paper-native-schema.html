The Paper-Native Dialog Schema

This structure exactly mirrors the Paper Dialog API builder pattern.

JSON Preset Example
{ "type": "dialog", "id": "player_picker", "title": "Select Target", "base": { "body": [ { "type": "plain_message", "content": "Who should receive the item?" }, { "type": "item", "material": "DIAMOND", "amount": 1 } ], "inputs": [ { "label": "Amount", "type": "number", "constraints": [1, 64] } ] }, "dialog_type": { "type": "multi_action", "columns": 3, "actions_source": "tab_completion", "exit_action": { "label": "Cancel", "tooltip": "Close without saving" } } }
Maps directly to Paper API
Dialog.create(factory -> factory.empty() // 1. Builds the DialogBase .base(DialogBase.builder(title) .body(List.of( DialogBody.plainMessage(msg), DialogBody.item(itemStack) )) .inputs(List.of( DialogInputBuilder.buildNumber(...) )) .build()) // 2. Applies the DialogType .type(DialogType.multiAction( tabCompletions, exitAction, 3 // columns )) );

Confirmation Type Variant

If the dialog_type is changed to confirmation, the schema swaps the multi-action keys for the exact parameters Paper expects:

"dialog_type": { "type": "confirmation", "confirm_action": { "label": "Accept" }, "cancel_action": { "label": "Decline" } }
