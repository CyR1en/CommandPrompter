How expressive do we want the inline syntax to be?

Currently, tags look like <d:tab[5]:Player>. To add layout control (titles, columns, exits), we have a few paths.

A

Key-Value Attributes (Inline)

Expand the bracket syntax to support named parameters.

/tp <d:tab[cols=3, title="Select Player", exit="Cancel"]:Player>
+ Great for quick, one-off commands
- Gets very long and hard to type in Minecraft chat
B

Preset-Driven (Config Files)

Move layout design into JSON/YAML, keeping commands tiny.

/tp <@player_picker>
+ Cleanest commands, unlimited design potential
- Requires server admins to edit files outside the game
C

The Hybrid Approach

Keep simple args inline, but allow preset overrides.

/tp <@player_picker[cols=4]:Player>
+ Best of both worlds
- Most complex to parse and implement
