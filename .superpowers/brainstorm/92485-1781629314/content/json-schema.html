How should we structure the JSON schema?

Since Paper API separates "inputs" (text, numbers) from "multi_action grids" (buttons), how should the JSON config model this?

A

Unified Dialog Model

Keep everything under "type": "dialog", expanding it with layout options and a TAB row type. This requires us to mix inputs and action grids in one screen.

{ "type": "dialog", "id": "punish_ui", "title": "Punish Player", "body": ["Select an action below:"], "layout": { "columns": 3, "exit_button": "Cancel" }, "rows": [ { "label": "Player", "input_type": "TAB" }, { "label": "Reason", "input_type": "TEXT" } ] }
B

Distinct Layout Types

Create a dedicated "type": "multi_action" for button grids, keeping the standard "dialog" strictly for text/number/choice inputs. (Reflects Paper's UI limitations).

{ "type": "multi_action", "id": "player_picker", "title": "Select Player", "body": "Pick a player:", "columns": 3, "exit_button": "Cancel", "source": { "type": "tab_completion" } }
