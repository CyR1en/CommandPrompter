The schema now explicitly supports the actions array for static buttons, alongside the actions_source helper.

JSON Preset: Static + Dynamic
{ "type": "dialog", "id": "gamemode_picker", "title": "Select Gamemode", "base": { "body": [ { "type": "plain_message", "content": "Choose a gamemode or use tab for custom:" } ] // inputs array omitted (optional) }, "dialog_type": { "type": "multi_action", "columns": 2, // 1-to-1 API Mapping: Static Buttons "actions": [ { "label": "Survival", "tooltip": "Standard gameplay", "return": "0" // The answer text (optional) }, { "label": "Creative", "return": "1" } ], // Helper Key: Dynamic Buttons "actions_source": "tab_completion", "exit_action": { "label": "Cancel" } } }
Schema Behaviors
The ActionButton Model: Each button object requires a label. • tooltip (optional): Displays on hover. • return (optional): The string sent to the command when clicked. If omitted, the label is used as the answer. Combining Static and Dynamic: If both actions and actions_source are defined, the system will render the static buttons first, then append the dynamic tab-completions after them in the grid. Validation Rules: Paper API requires at least 1 action. The schema will validate that either the actions array is non-empty, OR an actions_source is provided.
